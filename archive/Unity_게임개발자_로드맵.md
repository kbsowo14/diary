# 🎮 유니티 게임 개발자 로드맵

> 완전 초보부터 취업/출시까지, 단계별로 정리한 학습 가이드

---

## 🗺️ 전체 흐름 요약

```
1단계: 기초 체력 → 2단계: C# 프로그래밍 → 3단계: Unity 기초
→ 4단계: 2D 게임 개발 → 5단계: 3D 게임 개발 → 6단계: 심화 시스템
→ 7단계: 포트폴리오 → 8단계: 취업 또는 출시
```

---

## 1단계: 기초 체력 다지기 (2~3주)

게임 개발에 바로 뛰어들기 전에 알아두면 좋은 기본 지식입니다.

**컴퓨터 기초**
- 파일/폴더 구조 이해, 터미널(명령 프롬프트) 기본 사용법
- Git & GitHub 기초 (버전 관리는 처음부터 습관 들이세요)

**수학 기초 (겁먹지 마세요)**
- 좌표계, 벡터의 개념 (방향과 크기)
- 삼각함수 기초 (sin, cos — 회전에 사용)
- 기초 물리 개념 (속도, 가속도, 중력)

**추천 자료**
- Khan Academy 수학 강좌 (무료)
- "Git 입문" 관련 유튜브 한국어 영상

---

## 2단계: C# 프로그래밍 배우기 (4~6주)

Unity의 스크립팅 언어는 C#입니다. Unity를 켜기 전에 C#의 기본을 먼저 익히세요.

**필수 학습 내용**
- 변수, 자료형 (int, float, string, bool)
- 조건문 (if/else, switch)
- 반복문 (for, while, foreach)
- 배열, 리스트 (List)
- 함수(메서드) 만들기와 매개변수
- 클래스와 객체 (OOP 기초)
- 상속, 인터페이스
- 접근 제한자 (public, private, protected)
- 열거형 (enum)
- 이벤트와 델리게이트 (기초 수준)

**추천 학습 방법**
- Visual Studio 또는 Visual Studio Code 설치
- 콘솔 프로젝트로 간단한 프로그램 만들어보기 (계산기, 숫자 맞추기 게임 등)
- 인프런, 유데미(Udemy)의 "C# 기초" 강좌

---

## 3단계: Unity 엔진 기초 (3~4주)

이제 Unity를 설치하고 엔진에 익숙해질 차례입니다.

**환경 설정**
- Unity Hub 설치 → LTS(장기 지원) 버전 설치
- Visual Studio와 Unity 연동 설정

**Unity 에디터 익히기**
- Scene / Game / Inspector / Hierarchy / Project 창의 역할
- GameObject와 Component 구조 이해
- Transform (위치, 회전, 크기)
- Prefab 개념과 사용법
- 에셋 임포트 방법

**스크립팅 기초 (Unity + C#)**
- MonoBehaviour 생명주기: Awake → Start → Update → FixedUpdate
- GetComponent, Find, Instantiate, Destroy
- Input 시스템 (키보드, 마우스 입력 받기)
- Rigidbody와 Collider를 이용한 물리 처리
- Trigger와 Collision 이벤트

**첫 번째 실습 프로젝트**
- 🎯 공 굴리기 게임 (Roll a Ball) — Unity 공식 튜토리얼 추천

---

## 4단계: 2D 게임 개발 (4~6주)

3D보다 2D가 구조가 단순해서 먼저 배우기 좋습니다.

**핵심 학습 내용**
- Sprite, Sprite Renderer, Sorting Layer
- 2D 물리: Rigidbody2D, Collider2D
- 2D 애니메이션: Animator, Animation 클립
- Tilemap으로 맵 만들기
- 카메라 따라가기 (Cinemachine 2D)
- UI 시스템 기초: Canvas, Text, Button, Image
- Scene 전환 (SceneManager)
- PlayerPrefs로 간단한 데이터 저장

**실습 프로젝트 (택 1~2)**
- 🐦 Flappy Bird 클론
- 👾 2D 플랫포머 (점프, 적, 아이템)
- 🚀 2D 슈팅 게임 (총알 발사, 적 패턴)

---

## 5단계: 3D 게임 개발 (6~8주)

2D에서 익힌 개념을 3D로 확장합니다.

**핵심 학습 내용**
- 3D 모델, 머티리얼, 텍스처, 셰이더 기초
- 3D 물리: Rigidbody, 다양한 Collider
- 캐릭터 컨트롤러 (CharacterController 또는 직접 구현)
- 3D 애니메이션: Mecanim, Animator Controller, Blend Tree
- 라이팅 기초: Directional Light, Point Light, Baked vs Realtime
- 내비게이션: NavMesh로 AI 경로 탐색
- Cinemachine 3D 카메라
- Raycast 활용 (총 쏘기, 클릭 감지 등)
- 파티클 시스템 (이펙트)
- 오디오: AudioSource, AudioClip, Mixer

**실습 프로젝트 (택 1~2)**
- 🏃 3D 러너 게임
- 🔫 FPS 슈팅 게임 (기본)
- 🗡️ 3인칭 액션 게임

---

## 6단계: 심화 시스템 설계 (6~10주)

실제 게임에 필요한 시스템들을 배웁니다. 여기서부터 "진짜 게임 개발자"가 됩니다.

**프로그래밍 심화**
- 디자인 패턴: 싱글톤, 옵저버, 상태(State), 오브젝트 풀링, 팩토리
- ScriptableObject 활용 (데이터 관리)
- 코루틴(Coroutine)과 async/await
- 이벤트 기반 설계 (UnityEvent, C# event)

**게임 시스템**
- 인벤토리 시스템
- 대화(다이얼로그) 시스템
- 퀘스트 시스템
- 세이브/로드 (JSON 직렬화, 파일 저장)
- 게임 매니저, 씬 관리 아키텍처
- 오브젝트 풀링 (성능 최적화)

**UI/UX 심화**
- 메인 메뉴, 설정, 일시정지 화면
- 체력바, 미니맵, 인벤토리 UI
- DOTween 등을 이용한 UI 애니메이션

**성능 최적화**
- Profiler 사용법
- 드로우콜, 배칭, LOD
- 메모리 관리, 가비지 컬렉션 줄이기
- 에셋 번들 / Addressable 기초

## 8단계: 포트폴리오 프로젝트 (2~3개월)

취업이든 인디 출시든, 완성된 프로젝트가 가장 중요합니다.

**포트폴리오의 핵심 원칙**
- 미완성 10개보다 완성된 1~2개가 낫습니다
- "처음부터 끝까지" 만든 경험이 중요합니다
- 코드 품질, 구조, 주석에도 신경 쓰세요
- GitHub에 소스 코드를 공개하세요

**추천 포트폴리오 구성**
1. **메인 프로젝트** — 본인의 실력을 보여주는 완성도 높은 게임 1개
2. **서브 프로젝트** — 다른 장르나 기술을 보여주는 소규모 게임 1~2개
3. **기술 데모** — 특정 시스템(예: 인벤토리, AI, 셰이더)을 깔끔하게 구현한 것

**프로젝트 완성 체크리스트**
- [ ] 타이틀 화면, 게임 오버, 클리어 화면
- [ ] BGM과 효과음
- [ ] 설정(볼륨, 해상도 등)
- [ ] 빌드 후 실행 테스트
- [ ] itch.io 또는 Google Play에 배포

---

## 9단계: 취업 준비 또는 인디 출시

**게임 회사 취업 루트**
- 이력서 + 포트폴리오 PDF/영상 준비
- GitHub 코드 정리
- 기술 면접 대비: C#, 자료구조, 디자인 패턴, Unity 동작 원리
- 코딩 테스트 준비 (회사에 따라 다름)
- 게임잼 참여 이력이 있으면 플러스

**인디 게임 출시 루트**
- Steam, Google Play, App Store, itch.io 중 플랫폼 선택
- 스토어 페이지 세팅, 스크린샷, 트레일러 제작
- 커뮤니티 구축 (디스코드, 트위터/X, 유튜브)
- 마케팅 기초 (프레스킷, 위시리스트 모으기)
